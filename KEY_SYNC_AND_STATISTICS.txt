📋 FEATURE 1: KEY SYNCHRONIZATION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

How It Works:
1. Discord Bot generates key → saves to license_keys.json
2. Admin Dashboard API endpoint → reads from license_keys.json
3. Dashboard automatically loads keys on page load
4. Keys appear in Dashboard instantly ✅

📊 FEATURE 2: USAGE STATISTICS (WHO USED KEY & WHEN)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

What Gets Tracked:
✓ Discord username who checked the key
✓ Timestamp of when key was checked
✓ Status (ACTIVE/INACTIVE) at time of check
✓ Full history stored in key object

Where It Appears:
Admin Dashboard → Keys Tab → Each key now shows:
  📊 Usage History (Last 5 checks)
    👤 alice • 2026-04-17 10:30:45
    👤 bob • 2026-04-16 15:22:10
    👤 charlie • 2026-04-15 09:15:32
    (older entries...)

How Statistics Are Recorded:
1. User types: /checkkey STX-XXXXX-XXXXX-XXXXX-2026-04-17
2. Bot validates key
3. Bot automatically records:
   {
     "checkedBy": "alice",
     "checkedAt": "2026-04-17T10:30:45.000Z",
     "status": "✅ ACTIVE"
   }
4. Statistics saved to license_keys.json
5. Dashboard displays last 5 checks

A: Yes! All old keys continue to work and will show usage history.

Q: How long does sync take?
A: Instant! Dashboard fetches keys on page load and displays them.

Q: Can I still create keys in Dashboard?
A: Yes, but they won't have usage history until checked from Discord.

Q: How much history is stored?
A: All checks are stored (unlimited), but dashboard shows last 5.

Q: Does this work offline?
A: No, requires internet connection to sync and track.

Q: Can I export usage statistics?
A: Currently shown in dashboard. You can also read license_keys.json directly.
